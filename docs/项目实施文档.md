## TestReceiver（本地接收器）使用说明

目的：在本地验证心跳（TCP/UDP）能否到达目标，从而为离线验证和 CI 提供简单接收端。

位置：`src/TestReceiver`，已编译并发布为自包含 exe：`artifacts/TestReceiverPublish/TestReceiver.exe`。

运行（自包含 exe，推荐）：

```powershell
# 在仓库根执行：
.\artifacts\TestReceiverPublish\TestReceiver.exe
```

源码运行（需 .NET SDK）：

```powershell
dotnet run --project src/TestReceiver
```

说明：TestReceiver 在默认端口监听：TCP 8441、UDP 4565。若需要修改端口，请编辑 `src/TestReceiver/Program.cs` 并重新发布。

验证流程示例：
1. 启动 `TestReceiver`。
2. 运行 `SimulatorRunner`：

```powershell
.\artifacts\SimulatorRunnerPublish\SimulatorRunner.exe
```

3. 观察 `TestReceiver` 控制台输出是否收到来自 `SimulatorRunner` 的 HEARTBEAT 消息；同时查看 `logs/heartbeat-YYYYMMDD.log` 的汇总结果。

注意：若 `SimulatorRunner` 使用 `localhost` 导致 IPv6/IPv4 不匹配，建议使用 `127.0.0.1`（已在 Runner 中强制使用）。
