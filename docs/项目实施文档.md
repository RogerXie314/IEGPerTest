### 2026-03-03：v3.2.0 — NetworkStream 写锁（修复 3000 客户端离线 + 策略待发送）

**版本**：v3.2.0（`main` 最新）

#### 根本问题

v3.1.0 复用心跳 TCP stream 发送日志后，`HeartbeatWorker`（定时写心跳包）与 `LogWorker`（按 EPS 写日志包）**并发调用同一 `NetworkStream.WriteAsync`**。`NetworkStream` 不是线程安全的，并发写导致：

1. **包体字节互相穿插**：PT 包头/体错位，平台无法解析 → 主动断开连接 → **3000 客户端全部离线**。
2. **策略回包损坏**：`PolicyReceiveWorker` 通过 HTTPS 回报结果，但心跳 stream 读取/写入混乱 → 平台策略审计始终显示**"待发送"**。

#### 修复方案

`HeartbeatStreamRegistry` 新增每 clientId 写锁（`SemaphoreSlim(1,1)`）：

| 文件 | 变更 |
|---|---|
| `src/SimulatorLib/Workers/HeartbeatStreamRegistry.cs` | `_writeLocks` 字典；`Register` 时按需创建；`AcquireWriteLockAsync` / `ReleaseWriteLock` |
| `src/SimulatorLib/Workers/HeartbeatWorker.cs` | 心跳写入前获取锁，finally 释放 |
| `src/SimulatorLib/Workers/LogWorker.cs` | 共享 stream 写入前获取锁，锁等待期间重取最新 stream，finally 释放 |

## 待办（优先级）

2. 策略字段覆盖率验证：对照平台后端期望的全部 JSON 字段逐一确认。
