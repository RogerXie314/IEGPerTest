# 线程模型澄清

## 实际观察（2026-04-04）

### 测试场景
- 500个客户端启动心跳
- 100个客户端发送日志

### 线程数观察
- 启动心跳后：515个线程
- 启动日志后：仍然是515个线程

### 结论
**每客户端只有1个OS线程（心跳线程），日志发送复用心跳线程，不创建新的OS线程。**

计算：515 = 500（心跳线程）+ 15（系统线程）

## 与之前理解的差异

### 之前的错误理解
- 每客户端2个OS线程（心跳 + 日志）
- 日志线程独立创建：`AfxBeginThread((AFX_THREADPROC)ThreadFunc_MsgLogSend,...)`
- 500客户端 + 100日志 = 600个业务线程

### 实际情况
- 每客户端1个OS线程（心跳线程）
- 日志发送在心跳线程中完成，或者日志线程创建后很快退出
- 500客户端 = 500个业务线程
