## 三、技术设计

### 功能1：老版本注册（V300R006C02B090）

**背景**：当前 `RegistrationWorker` 硬编码 `clientVersion = "V300R011C01B030"`，
该版本客户端注册后，平台不允许对其执行白名单上传。
老版本 `V300R006C02B090` 平台侧仍支持白名单推送协议。

**改动范围**：
- `MainViewModel`：新增 `RegClientVersion` 属性，默认 `"V300R011C01B030"`
- `MainWindow.xaml`：注册设置区增加"版本"下拉（两项：新版/老版）
- `RegistrationWorker.RegisterAsync`：增加 `clientVersion` 参数，透传到 `BuildClientLoginJson`

---

### 功能2：白名单随机轮转上传逻辑

**现状**：`WhitelistUploadWorker` 每次取前 N 个客户端，每 run 一次就全部发一遍，没有去重、无轮转。

**新逻辑**：
```
AddTask 触发时：
  - 重置 已上传集合（uploadedSet）
  - 启动定时循环（间隔15分钟）

每轮循环：
  - 从 Clients.log 读取所有已注册客户端
  - 过滤掉 uploadedSet 中已上传的
  - 从剩余中随机选取 min(clientCount, 剩余数量) 台
  - 并发上传
  - 将本轮成功的客户端加入 uploadedSet

再次点击 AddTask：
  - 清空 uploadedSet（允许重新上传）
  - 重新开始循环
```

**改动范围**：
- `WhitelistUploadWorker`：重构为带状态的长生命周期 Worker
- `MainViewModel`：白名单任务独立 CTS，状态实时上报给 TaskPanel

---

### 功能3：策略下发接收与反馈

**背景**：平台通过已建立的 TCP 长连接向客户端推送策略（下行命令）。
客户端收到后需按协议回包（ACK/Result），平台才显示"策略已生效"。

**设计**：
- 新增 `PolicyReceiveWorker`（在 HeartbeatWorker 的 TCP 连接上复用接收逻辑）
- 在 TCP 接收循环里解析下行报文头，识别策略下发命令字
- 收到策略后，模拟延迟（500ms~2s随机）后发送 Result 回包
- 策略类型参考 `CmdWords.cs` 中已定义的命令字

**改动范围**：
- `HeartbeatWorker`：接收循环目前可能只丢弃下行包，需扩展为转发给 PolicyReceiveWorker
- 新增 `PolicyReceiveWorker.cs`
- `MainViewModel`：新增策略下发统计（收到/已回包）

---

### 功能4：统一任务面板（TaskPanel）

**模型**：新增 `TaskRecord`（位于 `SimulatorLib/Models/TaskRecord.cs`）

```csharp
public class TaskRecord : INotifyPropertyChanged
{
    public int    Id          { get; }       // 序号（自增）
    public string TaskType    { get; }       // "心跳" | "日志HTTPS" | "日志TCP" | "白名单上传" | "策略接收"
    public DateTime StartTime { get; }       // 开始时间
    public int    ClientCount { get; set; }  // 客户端数量
    public int    IntervalSec { get; set; }  // 间隔(秒)，0=不适用
    public string StatusText  { get; set; }  // 运行中/已停止/已完成
    public int    SuccessCount{ get; set; }  // 成功次数
    public int    FailCount   { get; set; }  // 失败次数
    public string Detail      { get; set; }  // 补充信息（如当前轮次、剩余客户端等）
}
```

**UI**：
- 右侧面板底部增加 `DataGrid`，绑定 `ObservableCollection<TaskRecord>`
- 列：序号 | 类型 | 开始时间 | 客户端数 | 间隔 | 状态 | 成功/失败 | 详情
- 每次"添加任务"自动新增一行，"停止/结束任务"更新对应行状态
- 使用颜色区分状态（绿=运行中，灰=已停止，橙=异常）
